## 회원가입 로직
1. POST Method를 이용하여 닉네임, 이메일, 비밀번호를 전달 받음
(이 때, 프론트 단에서 비밀번호는 비밀번호 input 방식을 이용. 비밀번호 확인 또한 프론트에서 진행)

2. 중복 가입 방지
- DB에 동일한 이메일이 존재하는지 여부를 확인하여 중복 가입 방지
(중복 시 Error throw)

3. DB에 데이터 추가
- DB에 데이터 INSERT. 단, 비밀번호의 경우 암호화 된 비밀번호를 저장하도록 함

4. 데이터 성공적으로 등록 완료 시 등록한 닉네임 및 이메일 Data Return

5. (선택) 만약 회원가입과 동시에 자동 로그인이 되도록 하고 싶다면 JWT Token 발급 및 해당 토큰 및 정보 세션 저장 과정 필요
